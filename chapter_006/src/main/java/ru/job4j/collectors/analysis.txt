1. Анализ времени работы различных типов сборщиков мусора рпи созданиии 100000 объектов без ссылок
    и ограничении памяти до 4 мб.

    Serial GC - среднее время работы ~ 6200 мс
    Parallel GC - среднее время работы ~ 10000 мс
    CMS GC - среднее время работы ~ 5500 мс
    CMS GC c установкой доступных потоков CPU - среднее время работы ~ 5500 - 8000 мс
    (в обратной зависимости от количества доступных логических ядер CPU)
    G1 GC - среднее время работы ~ 5500 мс

    В данном приложении лучшим выбором стал CMS GC, он также показал самую мнинимальную нагрузку на CPU.

2. Для приложения заявок из части 2 подойдет Serial GC или Parallel GC, т.к. приложение консольное, работает с одним
   пользователеми и не требует малых пауз при выполнении. Также подойдет CMS GC.

3. Для серверного приложения подойдет CMS GC (требуется малое количество и длительность пауз Stop the world).
    Также подойдет G1 GC, по документации он должен работать на мультипроцессорных системах с
    с большим количеством ОЗУ.
