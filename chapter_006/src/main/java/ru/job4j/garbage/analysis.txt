1. Размер объекта класса User:
    8 байт - заголовок класса
    40 байт - String name + по 2 байта на каждый символ в строке + возможные байты выравнивания для кратности 8
    16 байт - Integer age.
    В таком случае new User(null, null) займет в куче 64 байта.

2. Пустой объект без полей класса будет занимать лишь место заголовка - 8 байт.

3. В ходе работы программы было выставлено ограничение доступной памяти до 4 мб.
   В main методе в цикле создаются и выводятся в консоль объекты класса User.
   При низком уровне памяти система сама вызывает GC, который находит и помечает все созданные в цикле объекты пригодными
   для удаления, т.к. на них не ведут никакие ссылки в программе. У таких объектов вызывается метод finalize(), после чего
   объект удаляется из кучи.

   Также можно добиться выбрасывания исключения java.lang.OutOfMemoryError, если эти объекты не просто выводить на
   консоль, а также добавлять в список типа List.
